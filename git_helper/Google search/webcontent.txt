    <div class="post-text" itemprop="text">
<p>See the 'non-fast forward' section of '<a href="http://schacon.github.com/git/git-push.html">git push --help</a>' for details.</p>

<blockquote>
  <p>You can perform "git pull", resolve potential conflicts, and "git push" the result. A "git pull" will create a merge commit C between commits A and B.</p>
  
  <p>Alternatively, you can rebase your change between X and B on top of A, with "git pull --rebase", and push the result back. The rebase will create a new commit D that builds the change between X and B on top of A.</p>
</blockquote>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you "git pull" and it says "Already up-to-date.", and still get this error, it might be because one of your other branches isn't up to date.  Try switching to another branch and making sure that one is also up-to-date before trying to "git push" again:</p>

<p>Switch to branch "foo" and update it:</p>

<pre><code>$ git checkout foo
$ git pull
</code></pre>

<p>You can see the branches you've got by issuing command:</p>

<pre><code>$ git branch
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>When I got this error, I backed up my entire project folder.  Then I did something like</p>

<pre><code>$ git config branch.master.remote origin
$ git config branch.master.merge refs/heads/master
</code></pre>

<p>...depending on your branch name (if it's not master).</p>

<p>Then I did <code>git pull --rebase</code>.  After that, I replaced the pulled files with my backed-up project's files.  Now I am ready to commit my changes again and push.</p>
